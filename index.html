O que são Skills do Claude?

Skills são pacotes de conhecimento especializado que você instala no Claude para dar a ele capacidades persistentes. Pense como extensões de browser — você instala uma vez e o comportamento muda permanentemente. Lançadas em 2025, as Skills resolvem o problema mais frustrante de trabalhar com IA: ter que repetir o mesmo contexto toda vez que abre uma nova conversa.

Imagine que você é designer e precisa que o Claude sempre siga as guidelines visuais da sua marca, use as cores certas, o tom de voz correto e conheça todos os produtos. Sem Skills, você cola isso em cada conversa. Com uma Skill, o Claude já sabe tudo isso automaticamente — permanentemente.

⚡
Skills ≠ System Prompts: System prompts são texto estático colado em cada request. Skills são pacotes estruturados com metadados, instruções, ferramentas e exemplos que o Claude carrega INTELIGENTEMENTE — apenas quando relevante, usando ~100 tokens para verificar e carregando o conteúdo completo só quando necessário. É a diferença entre um colaborador que lê o manual antes de cada tarefa vs um que internalizou o conhecimento.

- Quando precisar criar headlines ou CTAs

## Instructions (o coração da skill)
Instruções detalhadas de como agir...

## Examples

Output: [exemplo de saída esperada]

## Constraints
O que NUNCA fazer nesta skill...

O que é n8n e por que integrar com Claude?

n8n é uma plataforma de automação de workflows open-source e self-hosted — pense no Zapier, mas sem limitações, gratuito para auto-hospedar e com acesso total ao código. Quando você combina n8n com Claude, cria automações inteligentes que tomam decisões reais, não apenas movem dados de A para B.

⚡
Por que n8n e não Zapier? n8n permite lógica complexa, loops, condicionais, código JavaScript nativo e — mais importante — integração profunda com a API do Claude sem limitações artificiais de tokens ou chamadas.

// arquitetura_n8n_claude

FLUXO BÁSICO
Trigger → Processo → Claude → Ação

Exemplo:
Email recebido → Extrair texto → Claude analisa → Responde automaticamente

Formulário enviado → Dados do lead → Claude qualifica → Notifica vendedor

// node_claude_no_n8n

O n8n tem um node nativo do Claude (Anthropic) que você configura com sua API key. Os principais parâmetros são:

  • Model — escolha entre Haiku (rápido/barato), Sonnet (equilibrado) ou Opus (mais poderoso)
  • System Prompt — o contexto permanente da automação
  • Max Tokens — limite de resposta (impacta custo)
  • Temperature — 0 para respostas consistentes, 1 para criativas
  • Messages — histórico de conversa para automações multi-turno

Debug e Erros Comuns no n8n + Claude

// erros_frequentes

ERROR: 401 Unauthorized
Causa: API key inválida ou expirada.
Fix: Settings → Credentials → Anthropic → verificar chave
ERROR: 429 Rate Limit
Causa: muitas requests por minuto.
Fix: adicione Wait node (1000ms) entre chamadas ao Claude
ERROR: max_tokens exceeded
Causa: resposta maior que max_tokens configurado.
Fix: aumente max_tokens para 4096+ ou use streaming
ERROR: JSON parse failed
Causa: Claude retornou texto em vez de JSON.
Fix: adicione no prompt "responda APENAS em JSON válido, sem texto"

O que é Composio?

Composio é uma plataforma que conecta Claude a mais de 250 ferramentas e apps com uma única integração. Em vez de você construir cada integração manualmente via API, o Composio oferece conectores prontos, autenticação gerenciada e actions padronizadas que o Claude pode executar diretamente.

🔌
O diferencial: Com Composio, você fala "conecte ao GitHub, leia os issues abertos e crie um resumo de prioridades" — e Claude executa isso de verdade, sem você escrever uma linha de código de integração.

O que é o Model Context Protocol?

MCP é um protocolo aberto criado pela Anthropic que padroniza como modelos de IA se conectam a ferramentas, dados e serviços externos. É como uma "porta USB universal" para IAs — qualquer aplicativo que implementa MCP funciona com qualquer IA que suporta o protocolo.

🔗
MCP vs Skills vs Composio: MCP é infraestrutura de conexão em tempo real (como acessar um banco de dados ao vivo). Skills são pacotes de conhecimento estático. Composio é uma camada de abstração sobre APIs. Eles se complementam.

// arquitetura_mcp

O que é Extended Thinking?

Extended Thinking é um modo especial do Claude onde ele "pensa em voz alta" antes de responder — como ver o rascunho mental completo antes da resposta final. O Claude usa tokens extras para raciocinar, verificar, explorar alternativas e autocriticar antes de concluir.

🧠
Diferença prática: Sem Extended Thinking, o Claude dá uma boa resposta. Com Extended Thinking em problemas complexos, o Claude dá uma resposta que pode ser 40-60% mais precisa — especialmente em matemática, lógica, código e análise estratégica.

// como_ativar

ATIVAR VIA API
response = client.messages.create(
model="claude-sonnet-4-6",
max_tokens=16000,
thinking={
"type": "enabled",
"budget_tokens": 10000 # tokens para raciocinar
},
messages=[...]
)

# No Claude.ai: ícone de "raio" na caixa de mensagem
# Modo "Avançado" para thinking mais profundo

Quando Usar Extended Thinking?

// use_para

  • Problemas matemáticos e lógicos complexos — cálculos financeiros, otimizações
  • Decisões estratégicas — análise de cenários, planejamento de longo prazo
  • Código complexo — arquitetura de sistemas, debugging difícil
  • Análise de documentos longos — contratos com contradições, relatórios complexos
  • Criação de argumentos — quando precisar de raciocínio rigoroso e bem fundamentado

// nao_use_para

  • Perguntas simples e diretas — desperdiça tokens
  • Criação de conteúdo criativo — thinking não melhora muito
  • Tradução ou formatação — overhead desnecessário
  • Quando velocidade importa mais que precisão
⚠️
Extended Thinking custa mais tokens e é mais lento. Use estrategicamente — ative apenas quando a qualidade da resposta compensa o custo adicional.

Técnicas Avançadas de Extended Thinking

// adaptive_thinking

O modo Adaptativo (disponível no Claude.ai) deixa o Claude decidir automaticamente quanto pensar — ele aloca mais budget para problemas difíceis e menos para os simples.

Família de Modelos Claude

Claude Opus mais poderoso
Melhor para: análises complexas, estratégia, código crítico, Extended Thinking. Mais lento e caro — use quando qualidade > velocidade.
Claude Sonnet ⭐ recomendado
Melhor para: 80% dos casos. Equilíbrio ideal entre qualidade, velocidade e custo. Padrão para Claude Code e Claude.ai Pro.
Claude Haiku mais rápido e barato
Melhor para: alto volume, respostas simples, classificação, extração de dados, chatbots. Latência <1s em muitos casos.

Entendendo Tokens e Custos

Tokens são a unidade de medida do Claude. 1 token ≈ 0.75 palavras em inglês, ≈ 0.6 palavras em português (por ser mais longo). Você paga por input tokens (o que envia) + output tokens (o que recebe).

ESTIMATIVA DE TOKENS
# 100 palavras ≈ 130-150 tokens (português)
# Email curto: ~200 tokens
# Artigo de 1000 palavras: ~1300 tokens
# Contrato de 10 páginas: ~8000 tokens
# Codebase de 100k linhas: ~200.000+ tokens

import anthropic
client = anthropic.Anthropic()
# Contar tokens antes de enviar
count = client.messages.count_tokens(model="claude-sonnet-4-6", messages=[...])
print(f"Tokens: {count.input_tokens}")

O que são Agentes com Claude?

Agentes são sistemas onde o Claude toma decisões e executa ações em loop — sem precisar de intervenção humana a cada passo. Em vez de responder uma pergunta, o agente planeja, age, verifica resultados e decide o próximo passo.

// loop_do_agente

ARQUITETURA DO AGENTE
OBJETIVO
↓
Claude PLANEJA (quais steps para atingir o objetivo?)
↓
Claude AGE (usa tools: busca, código, APIs)
↓
Claude VERIFICA (o resultado foi o esperado?)
↓
Objetivo atingido? SIM → FIM
NÃO → volta para PLANEJA

// quando_usar_agentes

  • Tarefas com múltiplos passos interdependentes
  • Quando o caminho para o objetivo não é previsível
  • Processos que precisam de verificação e correção automática
  • Workflows que envolvem múltiplas ferramentas

Construindo seu Primeiro Agente

AGENTE COMPLETO EM PYTHON
import anthropic, json

client = anthropic.Anthropic()
tools = [
{"name":"buscar_web", "description":"Busca informações na web",
"input_schema":{"type":"object","properties":{"query":{"type":"string"}},"required":["query"]}},
{"name":"executar_codigo", "description":"Executa código Python",
"input_schema":{"type":"object","properties":{"code":{"type":"string"}},"required":["code"]}}
]

messages = [{"role":"user", "content":"Pesquise o preço do Bitcoin e calcule quanto vale 0.5 BTC em reais"}]

while True:
response = client.messages.create(model="claude-sonnet-4-6", max_tokens=4096, tools=tools, messages=messages)
if response.stop_reason == "end_turn": break
if response.stop_reason == "tool_use":
tool_call = next(b for b in response.content if b.type == "tool_use")
result = execute_tool(tool_call.name, tool_call.input)
messages += [{"role":"assistant", "content":response.content},
{"role":"user", "content":[{"type":"tool_result", "tool_use_id":tool_call.id, "content":str(result)}]}]
print(response.content[0].text)
